# Phase 3.1: AI Assistant Polish

## Summary

Make the existing AI assistant great: add iterative self-correction (render → see → adjust), multi-turn context management, suggested prompts, Cmd+K command palette, and natural language parameter adjustment.

**Effort estimate:** Large (3d+)

---

## 1. Iterative Self-Correction

### AI Orchestrator State Machine
Add `aiSlice.orchestrator` with states: `idle | thinking | running_tools | waiting_render | capturing_vision | proposing_followup | error`

### Loop Flow
1. AI applies graph-mutating tool calls (create/edit/connect/delete/set-param)
2. Orchestrator triggers render automatically
3. Capture viewer thumbnail (512px) + graph delta summary
4. Send result back to Claude as vision input
5. AI evaluates: satisfied → done; wants to adjust → propose change

### Loop Bounds
- `maxSelfCorrectionIters = 2` by default (configurable)
- Hard stop conditions: no graph delta since last render, same suggestion repeats, user says stop

### Render Triggering & Waiting
```typescript
requestAiRender(options: {
  viewerId: string;
  capture: { before?: boolean; after?: boolean; regions?: Region[] };
}) => Promise<RenderInspectResult>

interface RenderInspectResult {
  renderId: string;
  durationMs: number;
  images: { after: string; before?: string }; // data URLs
  graphRevision: number;
}
```

### What to Send Back to Claude
- User's original request (verbatim)
- Graph delta summary: nodes/params changed since last inspection
- Parameter values for touched nodes
- Current graph DSL (only if materially changed; otherwise just `graphRevision`)

### User Approval Gates
- **Default**: 1 automatic self-check pass; additional changes require confirmation
- Prompt: "AI wants to adjust further. Apply 1 more tweak?" → Apply / Stop / Open chat
- Destructive actions (delete nodes, disconnect branches): always gate

## 2. Multi-Turn Context Management

### Data Model
```typescript
interface AiSession {
  turns: AiMessage[];           // Full fidelity for UI
  summary: string;              // Running summary (model-generated)
  graphAnchors: GraphAnchor[];  // Periodic graph snapshots
  cost: TokenUsage;             // Running totals
  branches: BranchMarker[];     // When undo/redo changes graph
}
```

### Token Budgeting Strategy
Build a "send set" for each API call:
- **Always include**: system prompt, tool schemas, last 6-10 turns, latest summary, latest graph snapshot or summary
- **Near budget**: drop older turns first → compress tool outputs → reduce DSL size (send summary instead)

### Summarization
- When turns exceed threshold: call lightweight summarize request
- Summary captures: user goals, known graph structure, what's been tried + outcomes
- Replace pruned turns in send set with single summary message; keep full turns in UI

### Graph Snapshots in Context
- Send full DSL: at session start, after significant structural edits, after undo/redo branch
- Otherwise: send `get_graph_summary` (node types, key params) + `graphRevision`

### Session Persistence
- `localStorage` for small sessions; `IndexedDB` for sessions with images
- On reload: restore chat UI; validate against current engine graph revision

### Conversation Branching (Undo AI Action)
- When user undoes: append branch marker message to conversation
- "User undid actions up to revision X; current graph is Y."
- Next send set includes marker + current graph snapshot for model context

## 3. Suggested Prompts / Quick Actions

### Rule-Based Suggestion Engine
Inputs: current graph summary, selected node, viewer type, presence of key nodes
Output: 6-10 suggestion chips grouped by category:
- **Color**: "Make it warmer", "Increase contrast", "Lift shadows"
- **Effects**: "Add vignette", "Sharpen", "Film grain"
- **Compositing**: "Key out green", "Add mask"
- **Cleanup**: "Reduce noise", "Remove background"

### Update Triggers
- Graph revision change
- Node selection change
- NOT per-frame (too expensive)

### UI Placement
- Chips below chat input (fast, discoverable)
- Clicking a chip: executes deterministic action if possible, or prefills chat + runs AI

## 4. Cmd+K Command Palette

### UX
- Overlay palette (VS Code / Spotlight style)
- Global shortcut: Cmd+K (Mac), Ctrl+K (Windows/Linux)
- Search input with fuzzy matching
- Sections: "Graph", "Nodes", "Parameters", "AI"
- Keyboard navigation: ↑/↓, Enter to execute, Escape to dismiss

### Action Types
1. **Local deterministic** (no AI): create node by type, connect, focus viewer, undo/redo, quick param nudge
2. **AI single-shot**: "Ask AI to..." with minimal context (graph summary + selection only, no full conversation)

### State
- `cmdkSlice` or colocated in `aiSlice`
- Recent actions list
- Actions call the same tool executor used by chat for consistency

## 5. Natural Language Parameter Adjustment

### Target Selection Heuristic
1. If node selected → prefer its params
2. Else → prefer most recently AI-touched node
3. Else → ask clarifying question with clickable candidates

### Mapping Language to Deltas
Intensity mapping table (local, deterministic):
| Phrase | Delta |
|--------|-------|
| "a little" / "slightly" | +5-15% |
| "more" / "stronger" | +30-50% |
| "much" / "a lot" | +80-120% |
| "less" / "weaker" | -30-50% |

AI chooses: which parameter, direction (+/-), safe bounds from NodeSpec

### Confirmation UX
Inline "proposal card" in chat:
- "Increase Blur.radius: 5 → 12"
- Buttons: Apply / Edit value / Cancel
- Apply uses `set_param_relative` tool (consistent + undoable)

## 6. Vision Pipeline Improvements

### Multi-Capture Modes
- **Before/After**: capture viewer pre-change (cached) and post-change; send both
- **Region crops**: user drags rectangle on viewer; crops sent to AI for detail inspection
- **Compare strip**: generate single composite (left/right) locally to reduce image count

### Quality Tradeoffs
- Default: 512px for general use
- Escalate to 768-1024px when: user requests detail, or model asks for higher-res and user approves

### Annotation Overlay (Minimal)
- User markup: simple canvas overlay producing coordinates/rects
- Send annotation metadata alongside image (not heavy formats)

## 7. AI Tool Extensions

### New Tools

**`set_param_relative`**
```json
{
  "input": {
    "nodeId": "string",
    "param": "string",
    "mode": "add | mul | set",
    "value": "number",
    "clamp": { "min": "number", "max": "number" },
    "reason": "string"
  }
}
```
Safety: clamp required for high-impact params; reject NaN/inf; enforce bounds from NodeSpec.

**`render_and_inspect`**
```json
{
  "input": {
    "viewerId": "string",
    "capture": { "before": "bool", "after": "bool", "regions": "[Region]" },
    "include": { "dsl": "bool", "graphSummary": "bool", "touchedParams": "bool" }
  },
  "output": {
    "renderId": "string", "images": "...", "graphRevision": "number", "touchedParams": "..."
  }
}
```

**`get_graph_summary`**
Output: compact `{ nodes: [{id,type,label}], connections, viewers, keyParamsByNodeType }`

**`undo` / `redo`**
Input: `{ steps?: number }`. Safety: only within current session undo stack.

### Tool Execution
- Single executor in store validates args → applies via graph actions → returns structured result
- Never raw exceptions to the model

## 8. Cost & Performance

### Usage Tracking
- Store per-request: `{ inputTokens, outputTokens, usdEstimate }`
- Display: per-message cost, session total, toggle in settings to hide/show

### Rate Limiting & Debouncing
- Client-side: 1 in-flight AI request per session (queue additional)
- Backoff on 429/529 errors
- Debounce suggestion generation

### Streaming
- Stream assistant text into chat UI for responsiveness
- Buffer tool calls until model finishes the tool_use block, then execute

---

## 9. Error Handling

| Failure Mode | Response |
|---|---|
| API failure / rate limit | Inline error: "API error" + "Retry" button + "Copy debug info" |
| Context window exceeded | Auto-prune + retry once; if still too large, ask user to shorten |
| Invalid tool args / DSL ops | Reject locally, return tool error to model for self-correction |
| Render failure during loop | Break self-correction loop; show "Render failed — AI can't inspect" |
| AI service unavailable | Keep Cmd+K deterministic actions usable; show "AI offline" banner |
