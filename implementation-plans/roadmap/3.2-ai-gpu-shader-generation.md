## 3. Multi-Stage GPU Composition

### EffectRecipe (Multi-Stage Wiring)
```json
{
  "nodes": [
    { "local_id": "edge", "kernel_id": "edge_detect", "param_bindings": {} },
    { "local_id": "blur", "kernel_id": "gaussian_blur", "param_bindings": {} },
    { "local_id": "quant", "kernel_id": "color_quantize", "param_bindings": {} }
  ],
  "edges": [
    { "from": { "node": "edge", "port": "output" }, "to": { "node": "blur", "port": "input" } },
    { "from": { "node": "blur", "port": "output" }, "to": { "node": "quant", "port": "input" } }
  ],
  "outputs": { "final": { "node": "quant", "port": "output" } },
  "ui": { "group_title": "Oil Paint Effect", "collapsed_by_default": true }
}
```

### AI Decision: Split vs Single Shader
- **Single shader**: purely per-pixel color math + small sampling neighborhood (1-4 taps)
- **Multi-stage**: separable blur, edge detect + stylize, downsample/upsample, heavy sampling

### Intermediate Textures
- Reuse wgpu texture pool: allocate matching domain/format, release when downstream done
- Each stage declares I/O ports as usual in its KernelManifest

### Fusion Optimization (Optional)
Conservative "try fuse": only when both stages share identical domain/size, single-input single-output, second reads only output color. Generate temporary fused kernel in memory.

### UX
- Insert as collapsed group labeled with effect title
- Optional "pipeline view" toggle showing stage-by-stage previews

## 4. Shader Parameter Inference

### Location
Implement in Rust (shared core): `cascade-gpu/src/param_inference.rs`
Expose via WASM bridge + Tauri IPC: `infer_params(glsl_body) -> Vec<ParamSpecDraft>`

### Inputs
1. GLSL `uniform` declarations
2. Optional inline annotations:
```glsl
// @param intensity slider min=0 max=1 default=0.5 label="Intensity"
uniform float intensity;
```

### Heuristics (Deterministic Defaults)
| Name Pattern | Range | Default | UI Hint |
|---|---|---|---|
| `intensity/amount/strength/opacity` | [0, 1] | 0.5 | slider |
| `radius/size/scale` | [0, 100] | 5.0 | slider |
| `seed` | [0, 9999] int | 0 | slider |
| `angle/rotation` | [-180, 180] | 0 | slider |
| `color/tint` (vec3/vec4) | [0, 1] per channel | white | color_picker |
| `enabled/active` (bool) | â€” | true | checkbox |

### Validation
- Every `uniform` referenced has a param entry (except engine uniforms: time, resolution)
- Clamp defaults to [min, max]
- Reject NaN/inf, min >= max
- Ensure param count fits uniform buffer limits

## 6. Shader Safety & Sandboxing

### Static Safety Checks (Pre-Compilation Gate)
GLSL safety linter in Rust:
- Disallow `while` loops unless provably bounded (v1: disallow all `while`)
- Disallow `for(;;)` and `while(true)` patterns
- Limit max loop iterations (reject `for i < 4096` above threshold)
- Disallow forbidden ops (atomics, storage writes beyond expected)

### Runtime Resource Limits
- Cap dispatch size and workgroup sizes based on `wgpu::Limits`
- Memory budget: limit intermediate texture allocations per effect
- Cap max texture dimension for preview renders (512/1024)

### Runaway Shader Handling
- If preview dispatch times out / device-lost: recreate device, mark shader unsafe for preview
- Clear UI error: "Shader exceeded execution budget; simplify loops / reduce samples"

## 10. Error Handling

### Compile Errors (Line Mapping)
- When wrapping user GLSL with engine prelude, maintain `user_start_line` offset
- Convert naga error spans â†’ user line/col
- Structured error: `EngineError { kind: "ShaderCompileError", message, user_line, user_col, snippet }`

### Runtime Errors
- Device lost: surface `GpuDeviceLost`, attempt automatic recreation, re-render
- Unsupported features: detect at registration via `required_features` + `wgpu::Limits`

### Degradation
- GPU Script fails: show error on node, disable output (no silent behavior change)
- CPU fallback only if equivalent CPU node exists; otherwise fail loudly

## 11. Testing Strategy

### Effect Library Validation
- Load all bundled `*.effect.json` â†’ schema validate â†’ register kernels
- "Compile-only" test: naga parse/validate without GPU device
- Optional GPU smoke: render subset on machines with GPU

### Parameter Inference Tests
- Golden tests: input GLSL uniforms â†’ expected ParamSpec output
- Edge: ambiguous names, missing defaults, vec types, annotation overrides

### Safety Tests
- Static linter: reject infinite loop patterns, oversized loops
- Runtime: intentionally heavy shader â†’ verify timeout/device-lost â†’ proper error

### Cross-GPU Compatibility
- "Compat suite" of conservative effects (no exotic features)
- Run on: WebGPU (Chrome), Metal (macOS), DX12 (Windows), Vulkan (Linux) where available

### Key Considerations
- **Generators** (no image input): ensure KernelManifest supports empty inputs
- **Transitions** (two inputs + time): standardize port names (A, B, t)
- **Frame feedback** (true datamosh): requires persistent history buffers; keep Phase 3.2 spatial-only
