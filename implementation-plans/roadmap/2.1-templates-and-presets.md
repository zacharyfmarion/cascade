# Phase 2.1: Templates & Presets

## Summary

Implement a template system for loading pre-built graphs, ship 10-15 curated starter templates, add "Save as Template" from any graph, build a template browser UI, and support node group presets.

**Effort estimate:** Large (1-2 weeks including curated content + UI polish + tests)

---

## 1. Template Data Model

### TemplatePackage Schema (v1)
```json
{
  "schemaVersion": 1,
  "id": "cascade.template.film-lookdev.v1",
  "meta": {
    "name": "Film Look-Dev Starter",
    "description": "CDL + curves + grain + vignette with sensible defaults.",
    "author": "Cascade",
    "tags": ["film", "grade", "grain"],
    "category": "Look Dev",
    "createdAt": "2026-03-06T00:00:00Z"
  },
  "compat": {
    "minCascadeVersion": "0.8.0",
    "maxCascadeVersion": null,
    "requiredNodeTypeIds": ["std.cdl", "std.curves", "std.grain", "std.vignette"],
    "requiresGpu": false
  },
  "preview": {
    "thumbnail": { "kind": "asset", "path": "templates/thumbnails/film-lookdev.png" }
  },
  "inputs": [
    {
      "id": "inputImage",
      "kind": "image",
      "label": "Input image",
      "defaultAsset": "templates/samples/portrait.jpg",
      "bindings": [
        { "nodeSelector": { "tag": "TEMPLATE_INPUT_MAIN" }, "param": "source" }
      ]
    }
  ],
  "graph": { "projectJson": "<<<EXISTING SAVE/LOAD JSON>>>" }
}
```

### Storage
- **Bundled (shipped)**: `apps/web/public/templates/**` (works for web + Tauri, easy cache busting)
- **User-created (web)**: IndexedDB (JSON + thumbnail blob)
- **User-created (desktop)**: app data dir `templates/*.json` + `thumbnails/*.png`
- **Remote/community**: explicitly deferred to Phase 5.1

## 2. Template Loading

### Import Modes
- **New Project from Template**: Replace current project state entirely
- **Insert into existing graph**: Create nodes with fresh IDs and paste near cursor; return selection

### ID Remapping (Authoritative in Rust)
- Engine method takes graph JSON + import mode → returns `{ newGraph, idMap, warnings }`
- `idMap: { oldNodeId → newNodeId }` for all imported nodes, edges, I/O ports
- Frontend uses `idMap` to select/focus nodes post-insert

### LoadImage Node Handling
- Templates must NOT contain absolute file paths
- On "Save as Template": convert file-backed nodes into template inputs (source = null + binding)
- On instantiate: if `defaultAsset` exists, prefill it; otherwise prompt "Choose input image(s)"

## 4. "Save as Template"

### UX Flow
1. Project menu → "Save as Template..." (or Library panel → "+ New Template")
2. Metadata form: name, description, category, tags
3. Thumbnail: auto-render current viewer output at 512px wide → PNG
4. Sanitization: strip absolute file paths, convert to template inputs; strip API keys
5. Export: `TemplatePackage` JSON (+ thumbnail asset)

### What Gets Saved vs Stripped
- **Keep**: GPU Script GLSL code, node params, group structure, connections
- **Strip/Replace**: absolute file paths, OS-specific URIs, API keys, machine-specific state

## 6. Node Group Presets

### GroupPreset Schema (v1)
```json
{
  "schemaVersion": 1,
  "id": "cascade.grouppreset.edge-refine.v1",
  "groupTypeId": "std.group.edge_refine",
  "meta": { "name": "Soft Hair Edges", "tags": ["keying", "matte"] },
  "compat": { "minCascadeVersion": "0.8.0", "requiredNodeTypeIds": ["std.blur", "std.erode"] },
  "exposedParamDefaults": { "softness": 0.35, "erode": -0.02 },
  "groupGraph": { "groupJson": "<<<EXISTING GROUP EXPORT JSON>>>" },
  "preview": { "thumbnail": { "kind": "generated", "pngBase64": "..." } }
}
```

### How Presets Differ from Templates
- Group presets are scoped to a single group (not a whole project)
- Exposed parameters become preset controls (save/restore param combinations)
- Applied via "Apply preset" dropdown in the group editor

### UX
- In group editor: "Save preset..." button + preset dropdown to apply
- Versioning: `groupTypeId` + `schemaVersion` match; mismatch shows warning

## 7. Template Distribution & Compatibility

- Bundled templates version-pinned via `compat.minCascadeVersion/maxCascadeVersion`
- User templates load with warnings if version outside range; still attempt import
- Unknown fields ignored (forward-compatible)
- `schemaVersion` gates major format changes

## 8. Interaction with Other Roadmap Items

- **Share as URL (2.2)**: Same import pipeline; URL payload can be `TemplatePackage` JSON
- **AI assistant (3.1)**: AI can recommend templates by tags/categories; can prefill template inputs
- **GPU Script in templates**: Allowed; mark templates with `requiresGpu` if they contain GPU Script nodes
- **Node groups in templates**: Templates can contain groups; group presets are just group exports with metadata
- **Batch rendering (2.4)**: CLI can accept template + input override to render from templates

---

## 9. Error Handling

| Failure Mode | Response |
|---|---|
| Missing node types | Create **placeholder "MissingNode"** entries; graph loads but evaluation shows targeted errors |
| Version mismatch | Non-blocking warning + "Try anyway" (default) |
| Corrupt template JSON | Fail fast: "Template file invalid: missing `graph.projectJson`" |
| GPU required but unavailable | Load template; mark GPU Script nodes as error state; non-GPU nodes work fine |
| Template input not resolved | Graph loads with placeholder; node shows "Missing input — select a file" |

## 10. Testing Strategy

### Rust (core/engine)
- Import/remap test: load template graph → verify node/edge counts preserved and IDs differ
- Round-trip: project save → wrap as template → import → save again → compare (ignoring IDs/order)
- Compatibility warnings: missing node type triggers placeholder + warning array

### Frontend
- Unit: filter/search/tag logic, template registry loading, metadata validation
- E2E (Playwright): open Templates tab → pick template → resolve image prompt → graph appears → thumbnail shown in browser → save as template → re-open from user templates

### Content Validation
- All 12 bundled templates: load → verify no errors → render → verify output is non-empty
- Template round-trip: save → load → verify identical graph structure
