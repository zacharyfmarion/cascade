# Phase 1.1: Node Group Stability

## Summary

Audit and fix existing group bugs, stabilize enter/exit editing flow, ensure groups work with all node types (including GPU Script), add group import from JSON, and enable group duplication/instantiation within a project.

**Effort estimate:** Large (1-2 weeks)

---

## 1. Bug Audit Strategy

### Systematic Test Matrix
- **Connection operations**: Connect to/from group I/O nodes; type mismatches between group interface and inner nodes; dynamic port addition/removal while connections exist
- **Undo/redo**: Undo within groups; undo past the point of entering a group; undo group creation; undo port addition/removal
- **Nested groups**: Groups containing groups; evaluation order; error propagation
- **GPU Script nodes**: Inside groups; GLSL compilation context; parameter passing through group I/O
- **Edge cases**: Empty groups, single-node groups, groups with only I/O nodes, groups with no connections to outer graph

### Test Scenarios to Run
1. Create group → enter → add nodes → connect → exit → verify connections
2. Create group → undo → redo → verify state consistency
3. Nested group: create group A → enter → create group B → enter B → add nodes → exit B → exit A → evaluate
4. GPU Script node in group → modify GLSL params → verify params flow through group I/O
5. Dynamic port add/remove while connections exist → verify no dangling connections
6. Delete group node → verify all inner state cleaned up

---

## 2. Connection Handling Fixes

### Edge Cases
- **Type mismatches**: When group I/O port type doesn't match connected inner node type — need clear error surfacing
- **Dynamic port changes**: Adding/removing ports on group_input/group_output while external connections exist — must disconnect cleanly and notify user
- **Connection validation**: Ensure type-checking works across group boundaries (outer connection type must be compatible with inner connection type)
- **Multiple connections**: A single group output port driving multiple downstream nodes

### Code Changes
- `cascade-core`: Audit group connection validation in graph module; ensure type-checking traverses group boundaries
- `graphSlice`: Ensure `addEdge`/`removeEdge` handles group I/O nodes correctly
- Add `validateGroupConnections(groupId)` helper that checks all inner↔outer connection type compatibility

---

## 3. Undo/Redo within Groups

### Key Questions & Design
- **Undo past group entry**: If user enters group, makes changes, then undoes past the entry point — should exit the group automatically and restore pre-entry state
- **Per-group undo stacks**: Consider keying undo stacks by editing context (root graph vs group being edited) in `kernel.ts`
- **Undo of group creation**: Must clean up all inner nodes, connections, and I/O nodes
- **Undo of port addition/removal**: Must restore connections that were broken by the port change

### Implementation
- `kernel.ts`: Key undo/redo stacks by `editingStackPath` (current group context)
- Swap active undo stack when entering/exiting groups
- Group creation undo: single compound action that removes all inner state
- Port operations: store affected connections in the undo entry for restoration

---

## 4. Nested Group Edge Cases

### Evaluation Order
- Pull-based evaluator already handles nested evaluation through group I/O nodes
- Ensure cache keys include the full nesting path (group A → group B → node N)
- Dirty propagation must traverse group boundaries correctly

### Error Propagation
- Errors in deeply nested nodes must bubble up with full path context: `GroupA > GroupB > NodeN: error message`
- Per-node error attribution must work across nesting levels

### UI Breadcrumb Navigation
- Breadcrumb trail showing editing stack: `Root > Group A > Group B`
- Click any breadcrumb to navigate to that level
- Ensure viewport state (zoom/pan) is saved per nesting level

---

## 5. GPU Script Node Compatibility

### Within Groups
- GLSL compilation context should be independent of group nesting (shaders compile the same regardless)
- Parameter passing: GPU Script uniform params must flow correctly through group I/O port binding
- Hot-reload of GLSL within a group must trigger re-evaluation of the group node in the outer graph

### Testing
- GPU Script node inside group → modify GLSL → verify compilation and rendering
- GPU Script params exposed through group I/O → verify slider changes propagate to shader uniforms

---

## 6. Group Import from JSON

### Schema Validation
- Validate against `GroupPackage` schema on import
- Check `schemaVersion` for compatibility
- Verify all required node type IDs exist in the current registry

### ID Remapping
- Imported node IDs must not collide with existing graph — generate fresh IDs for all imported nodes, edges, and I/O ports
- Maintain an `idMap: { oldId → newId }` for connection rewiring
- Remap all internal connections using the idMap

### Connection Rewiring
- After ID remapping, reconnect all internal edges using new IDs
- External connections (group I/O to outer graph) must be wired by the user post-import

### Error Handling
- Corrupt/incompatible files: structured error with specific issue (missing fields, unknown node types, invalid connections)
- Missing node types: create placeholder "MissingNode" entries with stored original params; show targeted warnings
- Version mismatch: non-blocking warning + "Try anyway" default

---

## 7. Group Duplication/Instantiation

### Deep Clone Semantics
- Duplicate creates a fully independent copy — editing one does not affect the other
- All inner nodes, connections, I/O nodes get fresh IDs
- Parameters are copied by value

### Implementation
- `duplicateGroup(groupId)` → deep clone inner graph → remap IDs → insert as new group node
- Position duplicated group near the original

### Future Consideration
- Shared group definitions (edit one, update all) is a separate feature for a future "component" system — explicitly out of scope for Phase 1.1

---

## 8. UX Considerations

### Visual Indicators
- Collapsed group view: show group name, port labels, and a preview indicator
- Port labels on group nodes derived from group_input/group_output port names
- Visual differentiation between regular nodes and group nodes (subtle border/icon)

### Feedback
- Import/export: progress indicator for large groups; success/error toast
- Enter/exit: smooth transition animation; breadcrumb update
- Error states: group with missing node types shows warning badge; broken connections highlighted

---

## 9. Error Handling

### Failure Modes
- **Group creation with invalid selection**: Clear message about why selection can't be grouped
- **Connection to incompatible group port**: Type mismatch error with suggested fix
- **Import failure**: Structured error with actionable details (which nodes missing, which connections invalid)
- **Evaluation error in group**: Per-node error attribution with full nesting path
- **Undo/redo corruption**: Detect inconsistent state and offer "reset to last known good state"

### Surfacing
- Per-node error attribution via existing error system
- Toast notifications for import/export operations
- Inline warnings on group nodes with issues (badge icon + tooltip)

---

## 10. Testing Strategy

### Rust Unit Tests (`cascade-core`)
- Group creation/deletion round-trips
- ID remapping correctness
- Connection validation across group boundaries
- Nested group evaluation order
- Dirty propagation through groups

### Integration Tests
- Full pipeline: create group → add nodes → connect → evaluate → verify output
- Import/export round-trip: export → import → verify identical graph structure (modulo IDs)

### E2E Tests (Playwright)
- Create group from selection → verify UI state
- Enter/exit group → verify breadcrumb navigation
- Import group from JSON → verify nodes appear
- Undo/redo within groups → verify state consistency
- Drag-and-drop group operations

### Architectural Note
- GroupNode currently uses `as_any_mut()` downcasting — known architectural debt
- ResourceStore refactor (engineering roadmap) will provide a cleaner pattern
- Phase 1.1 should work within the current architecture, not block on ResourceStore
