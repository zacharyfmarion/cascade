# Phase 2.2: Share as URL / Embed

## Summary

Enable sharing Cascade graphs via URL links with compressed inline payloads. Add embed mode for iframes, read-only viewer mode, and "Remix" functionality to fork shared graphs.

**Effort estimate:** Medium (1-2 days) for core share/view/remix; Large (3d+) with robust asset inlining, shader safety, and cross-browser embed validation.

---

## 1. URL Encoding Strategy

### Approach: Hash-Based Inline Payload
- Store graph data in `location.hash` (avoids 2KB query string limits; keeps data out of server logs)
- URL format: `https://cascade.app/#share=v1.<compressed-data>`
- Mode/UI flags in query params: `?mode=embed&ro=1&controls=0`

### Compression
- **Library**: `lz-string` (`compressToEncodedURIComponent` / `decompressFromEncodedURIComponent`)
- **Payload**: Sanitized project JSON → lz-string compress → URL-safe string
- **Size limits**: Enforce max decoded payload size (e.g., 1MB); reject payloads with too many nodes or excessively long GLSL strings

### SharePayload Schema (v1)
```typescript
interface SharePayload {
  v: 1;                          // Schema version
  kind: 'project' | 'template'; // What's being shared
  project: ProjectJson;          // Sanitized graph data
  assets?: InlineAsset[];        // Optional small inline images
  meta?: {                       // Optional attribution
    title?: string;
    author?: string;
    description?: string;
  };
}
```

### Versioning
- Prefix: `v1.` before compressed data
- Unknown version: show "Link created with newer Cascade" + offer best-effort import

## 2. Share Button UX

### Location
- Top app header (global share button)
- Opens a share modal

### Modal Contents
- **Copyable URL**: Full share link with copy button
- **Embed code**: `<iframe>` snippet with configurable options
- **Options**:
  - Include parameters (default on)
  - Include inline assets (default off — images increase URL size)
  - Safe mode for embeds (default on — disables untrusted GPU shaders)
- **Recipient preview**: "Preview as viewer" button opens read-only mode in new tab

### Future Additions
- QR code generation
- Share analytics (deferred)

## 3. Graph Loading from URL

### Boot Flow
1. App starts → parse `location.hash` and `location.search`
2. If `#share=v1.` prefix detected → show "Importing shared graph..." loading screen
3. Decode → validate schema (Zod) → sanitize → `store.importProject(project)`
4. On success: render graph in appropriate mode (edit/view/embed based on `?mode=`)
5. On failure: show error panel with "Open empty project" + "Copy error details"

### Compatibility
- Missing node types: import graph but mark unknown nodes as placeholders with error badges
- Version mismatch: attempt best-effort import; show warnings for unrecognized fields

## 4. Embed Mode

### URL Flags
```
?mode=embed    — Activates embed shell
&controls=0    — Hide parameter controls (default 1)
&preview=1     — Show output preview (default 1)
&ro=1          — Read-only (default 1 in embed)
&safe=1        — Disable untrusted GPU shader execution (default 1 in embed)
```

### What's Visible vs Hidden
- **Visible**: Node graph (zoom/pan), output preview, parameter values (if controls=1)
- **Hidden**: Menu bar, file operations, AI chat, settings, share button

### Responsive Sizing
- Embed respects iframe dimensions; canvas auto-scales
- Minimum dimensions: 400x300; below that show "Resize for best experience"

### Cross-Origin
- Set appropriate CSP headers: `frame-ancestors *` (or configurable allowlist)
- Optional `postMessage` API: "ready" event + "resize" notification (keep minimal)

---

## 5. Read-Only Viewer Mode

### Behavior
- `?mode=view` or `?ro=1`
- Disable mutations at the store layer (not just UI):
  - React Flow: nodes not draggable/connectable
  - Parameter controls: visible but disabled
- Keep pan/zoom enabled
- Engine evaluation enabled (output renders correctly)

### UI Additions
- "Open in Cascade" CTA button → navigates to full editor with same graph
- "Remix" button → forks into editable workspace (see §6)
- Attribution bar: shows `meta.author` and `meta.title` if present

### Performance
- Skip loading editor-only subsystems via route-level code splitting
- Lighter bundle for view mode

---

## 6. Remix Flow

### What "Remix" Does
1. Takes the shared graph payload
2. Creates a new local project (new project ID)
3. Optionally regenerates node IDs via re-import
4. Switches to full edit mode
5. Stores attribution: `{ sourceUrl, sharedAt, author, title }`

### What Gets Preserved vs Reset
- **Preserve**: graph structure, node params, GPU Script GLSL, connections
- **Reset/Strip**: API keys, local file paths, machine-specific state, project ID

---

## 7. Image/Asset Handling

### Strategy (v1)
- Support only external URLs already present in node params
- Show placeholders + "missing asset" warnings when external URLs fail to fetch
- GPU Script GLSL is text — travels with the graph automatically

### Optional Enhancement
- Allow inlining small images as data URLs in `assets[]`
- Strict caps: max 200KB per image, max 3 images total
- Toggle in share modal: "Include images" (default off)

### LoadImage Nodes in Shares
- If LoadImage references a local file path → strip on share (sanitization)
- If LoadImage references a URL → preserve (may fail for recipient)
- Show clear "Missing input" placeholder when image unavailable

## 8. Security Considerations

### Input Validation
- Treat share payload as untrusted: strict Zod schema validation
- Size limits: max nodes, max string lengths, max decoded bytes (prevent payload bombs)
- Never `dangerouslySetInnerHTML` for any shared strings

### GPU Script Safety
- `safe=1` (default for embeds): disable custom shader execution or require explicit user click-to-run + warning
- Show "Contains GPU Script" badge on shared graphs
- In safe mode: GPU Script nodes show "Shader blocked in safe mode" instead of rendering

### Secret Prevention
- Sanitize function strips: API keys, localStorage references, absolute file paths
- Never read `localStorage` keys into share payload
- Validation: assert no API key patterns in serialized output

## 9. Short URL Service (Future Foundation)

### Architecture
- Don't block v1: define `ShareSource = 'inline' | 'id'`
- Future: `#shareId=abc123` fetches payload JSON from a Cascade-hosted service
- Keep codec stable so server stores the same `SharePayload` object
- Server requirements: KV store, rate limiting, optional analytics

## 11. Error Handling

| Failure Mode | Response |
|---|---|
| Invalid/corrupt URL | Error panel: "This share link appears to be invalid" + "Open empty project" |
| Unsupported version | "Created with newer Cascade" + best-effort import attempt |
| Missing node types | Import with placeholder nodes + error badges + warning list |
| Asset load failures | Per-node "Missing asset" placeholder + warning toast |
| GPU unavailable (shared graph has GPU nodes) | GPU nodes show error; non-GPU nodes work fine |
| Payload too large | "This graph is too large to share via URL" — suggest file export |

## 12. Testing Strategy

### Unit Tests
- Codec round-trip: encode → decode → verify identical payload
- Size limit enforcement: verify rejection of oversized payloads
- Sanitization: verify API keys, file paths never present in output
- URL parsing: verify correct extraction of share data + mode flags

### Integration Tests (Playwright)
- Share → open viewer → verify graph renders
- Share → remix → verify editable graph
- Embed mode: verify layout, controls hidden/shown per flags
- Cross-browser: Safari, Chrome, Firefox URL/hash behavior + large payload handling

### Edge Cases
- Very long GLSL in GPU Script nodes → verify URL still works
- Share graph with 100+ nodes → verify compression handles it
- Share → load in older Cascade version → verify graceful degradation
